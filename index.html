반품·교환 Follow-Up
단계별 진행 현황을 관리하세요

클로드에게 효율적으로 업데이트 요청하는 법

⚠️ 클로드는 Netlify URL에 직접 접근하여 업데이트할 수 없어요. 항상 새 HTML 파일을 받아서 재업로드하는 방식으로 진행해요.

📋 Netlify 업데이트 워크플로우

STEP 01
클로드에게 요청
claude.ai 채팅에서 이 HTML 파일을 첨부하고 원하는 내용을 설명하세요. 파일 첨부가 핵심이에요 — 파일 없이 텍스트만 보내면 클로드가 기존 내용을 모릅니다.
STEP 02
새 index.html 다운로드
클로드가 업데이트된 index.html을 드립니다. 다운로드하세요.
STEP 03
Netlify에 재업로드
Netlify 대시보드 → Production deploys → 새 파일 드래그앤드롭. 30초 내 자동 배포됩니다.

✍️ 요청 템플릿 (복사해서 사용)

이 HTML 파일에 [추가/수정/삭제] 해줘: 추가: [내용 설명] 수정: [기존 내용] → [바꿀 내용] 삭제: [제거할 항목] 요청사항 예시: - "중국대행 탭에 반품불가 안내 문자 추가해줘" - "교환 탭에 6단계 '도매처 교환접수 완료' 스텝 추가" - "비용계산기에 이번달 메모 입력란 추가" - "FU 탭 반품 단계에 '마켓 승인 기한 D-3 알림' 추가"
💡 팁: 요청이 구체적일수록 정확하게 반영돼요. "이런 느낌으로" 보다 "A 위치에 B를 추가"가 훨씬 효과적입니다. 그리고 항상 최신 HTML 파일을 첨부해야 기존 내용이 유지돼요.

📁 파일 관리 팁

TIP 01
파일 버전 관리
업데이트 받을 때마다 이전 파일을 "index_20250115.html" 형식으로 날짜 붙여 보관해두면 롤백 가능해요.
TIP 02
로컬 + 넷리파이 병행
PC에서도 더블클릭으로 열 수 있어요. 인터넷 없어도 됨. 폰에서 쓸 때는 Netlify URL 북마크 사용.
TIP 03
데이터 주의사항
FU 케이스, 비용 데이터는 브라우저 localStorage에 저장돼요. 브라우저 데이터를 지우면 초기화됩니다. 중요한 데이터는 "내보내기"로 백업해두세요.
